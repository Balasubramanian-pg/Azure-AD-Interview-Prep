# SCHEMA

Introduction
Database schemas define the structure of data storage systems, including tables, relationships, and constraints. In the semantic web, schemas (e.g., Schema.org) provide standardized vocabularies for structuring metadata to improve machine understanding. In cognitive science, schemas are mental frameworks that help organize knowledge and predict experiences. This guide explores these domains, their core principles, and practical examples.

---

### Core Concepts

#### 1. **Database Schemas**
- **Purpose**: Define the logical structure of a database, specifying entities, attributes, and relationships.
- **Components**:
  - **Tables**: Represent entities (e.g., "Employees," "Departments").
  - **Columns / Fields**: Describe attributes (e.g., `employee_id`, `first_name`, `hire_date`).
  - **Primary Key**: Uniquely identifies a row (e.g., `employee_id`).
  - **Foreign Key**: Establishes relationships between tables (e.g., linking "Departments" to "Employees").
  - **Data Types**: Define acceptable values (e.g., `VARCHAR`, `DATE`, `INTEGER`).
  - **Constraints**: Enforce rules (e.g., `NOT NULL`, `UNIQUE`, `CHECK`).

- **Normalization**: Reduces redundancy and ensures data integrity (e.g., Boyce-Codd Normal Form).

#### 2. **Semantic Web Schemas**
- **Purpose**: Standardize metadata to enable machines to interpret content contextually.
- **Vocabulary**: 
  - Schema.org provides a shared vocabulary for entities like "Product," "Person," or "Event."
  - Markup syntaxes include JSON-LD, Microdata, and RDFa.
- **Example Use Cases**:
  - Rich Snippets in search results (e.g., star ratings, prices).
  - Enhanced social media previews.
  - Improving SEO through structured data.

#### 3. **Cognitive Schemas**
- **Purpose**: Mental frameworks that guide perception, learning, and decision-making.
- **Types**:
  - **Social/Behavioral Schemas**: Expectations about interactions (e.g., restaurant etiquette).
  - **Perceptual Schemas**: Patterns for interpreting sensory input (e.g., recognizing a face).
  - **Event Schemas**: Prototypical sequences (e.g., "going to a movie").
- **Role**:
  - Facilitate efficient processing of information by leveraging prior knowledge.
  - Support prediction and problem-solving.

### Examples

#### Database Schema Example (SQL)
```sql
CREATE TABLE Employees (
    employee_id INT PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    hire_date DATE,
    department_id INT,
    FOREIGN KEY (department_id) REFERENCES Departments(dept_id)
);
```

#### Semantic Web Schema (JSON-LD)
```json
{
  "@context": "https://schema.org/",
  "@type": "Product",

  "description": "High-quality wireless headphones with noise cancellation.",
  "price": "299.99",
  "priceCurrency": "USD",
  "brand": {
    "@type": "Brand",

  },
  "offers": {
    "@type": "Offer",
    "availability": "InStock",
    "seller": {
      "@type": "Organization",
      "name": "Tech Retailers"
    }
  }
}
```

#### Cognitive Schema Example
**Restaurant Schema**:  
- *Expectation*: A menu, table service, dining etiquette, and a check at the end.  
- *Adaptation*: If the restaurant uses self-service, diners update their schema to include selecting dishes at a kiosk first.
